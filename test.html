Test Your Pipeline

Upload a new Markdown file to the input bucket and verify the HTML appears in the output bucket.

1) Upload

# meeting notes

- Alice
- Bob
  1. Save as meeting.md.
  2. Upload to input bucket (notes-in-…).
  3. Wait a few seconds, then open the output bucket (notes-out-…).

2) Verify output

  • meeting.html exists in the output bucket.
  • index.html exists and lists the generated pages.
  • Download meeting.html and open it locally to view the rendered note.

3) Troubleshooting

No files in output?
  • Confirm the S3 event notification is on the input bucket with suffix .md.
  • Re-upload the file (modify name or content to retrigger).
Lambda error?
  • Open CloudWatch Logs → find your function’s log group/stream.
  • Check for exceptions (copy the first error line/stack trace if you need help).
Handler mismatch?
  • Lambda Runtime settings → Handler must be lambda_function.handler.
Wrong bucket names?
  • Ensure input ends with -in and output with -out to match code’s bucket swap.
