Create Two Buckets

We’ll use separate buckets for input and output. Keep all defaults (Block Public Access ON). Ensure both buckets are in the same Region and you stay in that Region for the whole lab.

Names

  • Input bucket: notes-in-<yourinitials+3digits> (e.g., notes-in-ad123)
  • Output bucket: notes-out-<yourinitials+3digits> (e.g., notes-out-ad123)

Steps

  1. Open Amazon S3 → Create bucket.
  2. Enter your input bucket name (e.g., notes-in-ad123), keep all defaults → Create bucket.
  3. Repeat for your output bucket (e.g., notes-out-ad123).

Upload a test Markdown file to the input bucket

  1. Create a local file named hello.md with content: 
    # hello

this is **bold** text.
  2. Upload hello.md to your input bucket (notes-in-…).

Do not upload anything to the output bucket manually.
